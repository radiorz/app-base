# 通用的规则

通用的规则也需要结合具体业务
分析业务得出具体规则


- 通话业务
  - 获取配置 同步配置
  - 远端/设备都可以配置设备的通话配置(通过元数据去限制)
  - 通话过程需要触发事件, 包括管理型的事件(可以用动作包括)
  - 
可以理解为
- 资源请求响应协议(CRUD+Sync(Client/Server))
- 事件(告警) 发出各种触发的事件 
- 事务 跟踪整个通话周期,


设备的资源绑定
- 包括人员的信息 人员通常也只有一份
  - 人员的绑定的资源 通常多份
- 属性配置 也可以类似资源 但通常只有一



基础的设备通常也包括

设备登入 设备登出 设备注销等
